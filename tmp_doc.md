## 1. 业务场景确认

### 1.1 内容发布场景

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        内容发布业务场景确认                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【内容来源】                                                               │
│  运营在后台手动创建内容（文案+图片/视频素材）                               │
│                                                                             │
│  【平台适配】                                                               │
│  系统自动适配不同平台的格式差异：                                           │
│  • Twitter：280字限制 + 话题标签 + 图片裁剪                               │
│  • 小红书：长文案 + emoji + 多图格式                                       │
│  • LinkedIn：专业语气 + 文章格式                                           │
│                                                                             │
│  【审核流程】                                                               │
│  内容必须经过人工审核才能发布，不可跳过：                                    │
│  运营创建 → 适配预览 → 运营审核 → 定时发布                                 │
│                                                                             │
│  【发布方式】                                                               │
│  定时发布（设置发布时间）或立即发布                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 客户咨询/线索获取场景

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    客户咨询/线索获取场景确认                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【客户来源】                                                               │
│  客户从各社交平台的评论/留言/私信中来：                                    │
│  • Twitter：评论、@提及、私信                                               │
│  • 小红书：笔记评论、@提及、私信                                            │
│  • LinkedIn：评论、私信、连接请求                                           │
│                                                                             │
│  【联系方式类型】                                                          │
│  客户可能留下以下联系方式之一：                                             │
│  • 电话号码（+65 xxxx xxxx）                                               │
│  • 邮箱地址（xxx@example.com）                                             │
│  • 社交账号ID                                                               │
│  • 官网入口（点击了某链接）                                                 │
│                                                                             │
│  【处理方式】                                                               │
│  1. Profile自动回复，友好互动                                              │
│  2. 从评论/留言中提取联系方式                                              │
│  3. 创建线索记录到后台                                                      │
│  4. 运营在后台跟进维护                                                      │
│                                                                             │
│  【进阶功能（可选）】                                                       │
│  • 自动发消息沟通细节（复杂，可后续实现）                                  │
│  • 人工在后台维护客户沟通记录（简单，先做这个）                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.3 飞书角色定位

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        飞书角色定位确认                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  飞书 = 运营内部协作工具，不对外                                           │
│                                                                             │
│  【飞书的使用场景】                                                         │
│  1. 收到新线索通知（后台推送到飞书群）                                     │
│  2. 查看Agent执行状态和日志                                                │
│  3. 向Agent发送指令（暂停、调整参数）                                      │
│  4. 接收异常告警通知                                                        │
│  5. 运营团队内部协作讨论                                                   │
│                                                                             │
│  【飞书不对外的场景】                                                       │
│  ❌ 客户不会直接在飞书咨询                                                  │
│  ❌ 飞书不是对外客服窗口                                                    │
│  ❌ 不会公开EA的飞书账号给客户                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
