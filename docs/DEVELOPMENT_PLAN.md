### Week 1-2：版本对比与导出（P0）

**目标：** 完善版本快照，实现历史版本对比和多格式导出

#### 后端
- [ ] `VersionSnapshot` 模型补充导出专用字段（export_data JSON，存储脱敏后的完整报价结构）
- [ ] 新增 `GET /api/versions/<id>/compare?v1=X&v2=Y` — 返回两个版本的字段级差异
- [ ] 重构 `exports.py` 的 Word 导出，支持指定版本号导出
- [ ] 新增 Excel 导出（python-docx + openpyxl），包含：封面页、项目信息表、物料明细表、费用汇总表、人力成本表
- [ ] 导出时自动附带版本号、导出时间、审核状态水印

#### 前端
- [ ] `VersionCompare.vue` — 版本列表页，新增「版本对比」按钮
- [ ] 对比视图：左右分栏或上下展开，高亮差异字段（名称/单价/数量/金额变化）
- [ ] 导出按钮组：PDF / Word / Excel 三按钮，下拉菜单选择版本
- [ ] 导出进度提示（后端大文件生成时）

#### 交付物

- 三种格式导出测试文件

### Week 3：人力工时管理·后端（P0）

**目标：** 工时记录写入报价单成本，支持按人员/模块统计

#### 数据模型
- [ ] `LaborHour` 模型：`module_id`, `user_id`, `date`, `hours`, `work_content`, `created_at`
- [ ] `LaborHourRate` 模型：`participant_type`（项目/机构/电气），不同类型不同工时单价

#### API 设计
- [ ] `GET /api/labor-hours?module_id=X` — 查询模块工时列表
- [ ] `POST /api/labor-hours` — 录入工时
- [ ] `PUT /api/labor-hours/<id>` — 修改工时
- [ ] `DELETE /api/labor-hours/<id>` — 删除工时
- [ ] `GET /api/labor-hours/summary?quotation_id=X` — 工时汇总（总工时、总人力成本，按人员分组）
- [ ] 工时数据计入 `OtherFee` 的人力费用类型，联动更新报价单总价

#### 权限
- [ ] 普通用户：仅能查看/编辑自己的工时
- [ ] 管理员/经理：可查看/编辑全部工时

#### 交付物
- 工时管理 API 文档
- Postman 测试集合

### Week 5：数据导入（P1）

**目标：** Excel 批量导入，减少人工录入成本

#### 功能
- [ ] 物料批量导入：`POST /api/materials/import`（Excel → 物料表，支持更新已有数据）
  - 模板下载
  - 导入预览（先预览再确认）
  - 导入结果报告（成功数/失败数/错误行）
- [ ] 用户批量导入：`POST /api/users/import`（从 SQL Server 同步以外，支持手动 Excel 导入）
- [ ] 工时数据导入：`POST /api/labor-hours/import`（支持月度工时表导入）
- [ ] 导入日志记录（OperationLog 记录每次导入）

#### 技术方案
- [ ] 使用 `openpyxl` 解析 Excel
- [ ] 后端导入任务超过 5 秒返回异步任务，轮询状态

### Week 6：报表统计（P1）

**目标：** 多维度报价数据统计分析

### Week 7：系统设置（P2）

**目标：** 支持企业自定义配置，降低运维成本

#### 功能
- [ ] **部门管理** — 从 SQL Server 同步的部门数据，支持手动维护显示顺序

- [ ] **审批流程配置** — 定义审批节点（如：提交→经理审核→财务复核→归档）

- [ ] **系统参数配置** — 默认税率、默认利润率、默认币种、汇率刷新频率
- [ ] **数据备份** — 手动触发 PostgreSQL 全量备份（pg_dump）

### Week 8：消息增强（P2）

## 五、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 报表查询慢，影响主业务 | 高 | 报表走只读副本库，或定时生成统计表 |
| Excel 导入格式兼容性问题 | 中 | 提供标准模板，限制格式（.xlsx only） |
| 邮件/钉钉推送到达率低 | 低 | 配置失败时降级为站内消息 |
| 国际化翻译质量 | 中 | 先用机器翻译，人工 Review 关键页面 |
| 测试覆盖不足，上线后 Bug 多 | 高 | Week 10 至少保证核心路径 100% 覆盖 |
