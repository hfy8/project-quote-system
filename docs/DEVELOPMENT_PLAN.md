## 四、人力工时域 ★★★

> 优先级：P0　｜　周期：Week 3-4

**新增功能域**，从零开发。

### 4.1 数据模型

```
LaborHour（工时记录）
├── id, module_id, user_id, date, hours, work_content, created_at
└── 关联：Module, User

LaborHourRate（工时单价）
├── id, participant_type（项目人员/机构人员/电气人员）
├── unit_price（每小时单价）
└── 关联：ParticipantType
```

### 4.2 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/labor-hours?module_id=X` | 查询模块工时列表 |
| POST | `/api/labor-hours` | 录入工时 |
| PUT | `/api/labor-hours/<id>` | 修改工时 |
| DELETE | `/api/labor-hours/<id>` | 删除工时 |
| GET | `/api/labor-hours/summary?quotation_id=X` | 工时汇总（总工时、总成本，按人员分组） |
| GET/POST/PUT/DELETE | `/api/labor-hour-rates` | 工时单价配置 |

**权限规则：** 普通用户仅能查看/编辑自己的工时；管理员/经理可查看/编辑全部。

**联动逻辑：** 工时汇总自动写入 `OtherFee`（费用类型=人力），联动更新报价单总价。

### 4.3 前端页面

| 页面 | 说明 |
|------|------|
| `LaborHours.vue` | 工时记录列表（日期范围、人员筛选） |
| `LaborHoursEntry.vue` | 工时录入/编辑表单 |
| `LaborHourRates.vue` | 工时单价配置（按参与类型） |
| `LaborHourReport.vue` | 工时报表（柱状图：每人工时对比；折线图：月度趋势） |
| `QuotationEdit.vue` 新增 Tab | 报价单内嵌「人力成本」Tab，展示工时汇总 |

- 工时管理完整页面截图（5 张）

- API 测试用例（Postman 集合）

## 五、数据管理域 ★★

- 物料可单个新增/编辑/删除

### 5.2 增强目标

#### Excel 批量导入（核心）
| 功能 | 说明 |
|------|------|
| 物料批量导入 | `POST /api/materials/import`，支持更新已有数据（按品名+规格匹配） |
| 用户批量导入 | 支持手动 Excel 导入（不依赖 SQL Server） |
| 工时数据导入 | 支持月度工时表导入 |

**导入流程：** 下载模板 → 上传 Excel → 预览数据（先确认再入库）→ 返回导入报告（成功数/失败数/错误行）

**技术要求：** 导入超过 5 秒返回异步任务，前端轮询进度；使用 `openpyxl` 解析；导入日志写入 `OperationLog`。

## 十二、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 报表查询慢影响主业务 | 高 | 报表走只读副本库或定时生成统计表 |
| Excel 导入格式兼容性 | 中 | 提供标准模板，限制 .xlsx 格式 |
| 邮件/钉钉推送到达率低 | 低 | 配置失败时降级为站内消息 |
| 国际化翻译质量 | 中 | 机器翻译 + 人工 Review 关键页面 |
| 测试覆盖不足上线后 Bug 多 | 高 | Week 10 核心路径 100% 覆盖 |
